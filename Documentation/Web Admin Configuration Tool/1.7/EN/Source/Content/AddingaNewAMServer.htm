Registering a New AM Server

You can also use the AM Servers tab in to register a new AM Server to the existing AM Servers a list of existing AM Servers in your Web Admin installation, register new AM Servers, and set a default AM Server for your SSP user logins.

Adding a New AM Server

To register a new AM Server with your existing Web Admin installation, do the following: 

On the AM Servers tab, click Register AM Server. The Register AM Server dialog opens.

Enter the following information: 

AM Server Name: Enter the name of the AM Server that you want to register.

Port Number: Enter the port number used for communication with the AM Server. Do not change the default port number on this screen.

Admin Username: Enter the username for the account of the AM Server Administrator.

Admin Password: Enter the password for the AM Server Administrator account.

If you want SSP users to connect to this AM Server by default, select the Make this AM Server the default for SSP user logins box. If you do not select a default server for SSP user logins, users are prompted to specify a AM Server on the login screen.

Click Save. If your AM Server registration is successful, a confirmation dialog opens to show the Unique Identifier of the AM Server. Click OK to close the dialog and return to the AM Servers tab. Details of the newly registered server are now visible in the AM Servers grid on this tab.
