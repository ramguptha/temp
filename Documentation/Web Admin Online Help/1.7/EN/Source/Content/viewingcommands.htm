Managing Queued Commands and Command History

You can view information about the commands that users have issued to mobile devices and computers within the following areas of the application:

Queued Commands: Contains information related to commands sent from that have not yet been carried out on the target devices and are still pending execution. A command may enter this queue because the target device is unavailable at the time the command is initiated.

Command History: Contains information related to commands sent from that have been completed (successfully or unsuccessfully) on the target devices. If there were any errors during the execution of a command, they show on this page.

You can also delete commands if you no longer want them to be listed on these pages:

Deleting a command from the command history simply removes it from the results grid.

Deleting a command from the queue not only removes it from the results grid, but may also prevent it from being carried out on the target device if the system has not yet pushed the command to the device.

To view queued commands or command history:

In the top left corner of the screen, click .

Click Queued Commands or Command History from the menu.

In the Device Type sidebar, select whether you want to view commands for Computers or Mobile Devices.

To quickly find specific commands within the results grid, you can use the search field to narrow the commands listed by specific criteria. Search results are updated dynamically as you type.

You may need to click your browser's refresh button to see updated information in the results grid.

To delete queued commands or command history:

In the top left corner of the screen, click .

Click Queued Commands or Command History from the menu.

In the Device Type sidebar, select whether you want to delete commands for Computers or Mobile Devices.

Select the checkbox of each command you want to delete.
